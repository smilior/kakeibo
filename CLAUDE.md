# プロジェクトルール

## 基本ルール
- すべての回答は日本語で行う
- コメントも日本語で書く

## Git運用ルール

### ブランチ戦略
- `main`: 本番用（常に動作する状態を維持）
- `feature/xxx`: 新機能開発
- `fix/xxx`: バグ修正
- `refactor/xxx`: リファクタリング

### 開発フロー
1. `main` から feature ブランチを作成
2. 小さい単位でこまめにコミット
3. 動作確認後、`main` にマージ
4. マージ後はブランチを削除

### コミットメッセージ
- `feat:` 新機能
- `fix:` バグ修正
- `refactor:` リファクタリング
- `docs:` ドキュメント更新
